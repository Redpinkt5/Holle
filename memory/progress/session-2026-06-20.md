# Session 2026-06-21（续）

## 用户反馈

用户在 Pet 中测试，AI 只能回复文字，不会实际播放。用户怀疑是不是因为模型只是 LLM、不能调用工具。

## 分析

Holle Music 的 AI 服务分两类：

1. **支持 function calling**：`DeepSeekService`、`ArkService` —— 有 `submit_tool_results`，会走 tool-calling 循环。
2. **纯文本服务**：`MiniMaxService`、`OpenAICompatibleService`（OpenAI / SiliconFlow / Kimi / Qwen / Zhipu 等）—— 原来的 `chat()` 只返回字符串，不走工具循环，所以确实“只能回复文字”。

如果用户通过 `/ai` 配置的 key 被探测为 MiniMax 或某个通用 OpenAI 兼容服务，就会出现只回复不播放的情况。

## 修复

为纯文本服务也加了本地播放兜底：

1. 在 `TUITools` / `AITools` 新增 `extract_play_query(text)`：去掉“播放 / 来一首 / 听 / 的 / 歌”等噪音词，得到歌手/歌名关键词。
2. 在 TUI 和 Pet 的纯文本分支里：
   - 如果用户消息包含播放意图词，提取关键词；
   - 调用 `search_local` 搜索本地歌单；
   - 有结果则调用 `auto_play_best_match` 自动播放最佳匹配。
3. 这样即使 AI 模型本身不调用工具，播放器也能根据用户话语执行播放。
